PhishingGuard Privacy Policy

Last Updated: 2025-11-29

1. Data We Collect

PhishingGuard collects only the minimum data required to detect phishing risks:

  • Client ID – a randomly generated identifier stored locally in your browser, used only to apply user-specific overrides and maintain consistent evaluations. This ID is never sent to external services.
  • Visited URL – the URL (or its normalized/hash form) that is sent securely to the PhishingGuard server for phishing analysis.

PhishingGuard does not collect any personally identifiable information (PII).

2. How We Use the Data

The collected URL is used solely for the following purposes:

  • Checking against known phishing and blacklist databases
  • Verifying safety using Google Safe Browsing API
  • Analyzing potential phishing risk using AI models (e.g., Gemini)

We do not use collected data for analytics, advertising, or profiling.

3. Data Storage

PhishingGuard does not store safe or non-threatening URLs.

Only the following types of data may be stored:

  • URLs (or URL hashes) confirmed as phishing threats
  • URLs explicitly reported by users
  • AI-generated explanations or detection reasons

We do not store personal data or detailed browsing history.

4. Data Sharing and External Services

PhishingGuard does not sell, share, or use any data for advertising, tracking, or analytics.

For phishing analysis, URLs may be transmitted to:

  • Google Safe Browsing API
  • AI analysis services (e.g., Gemini API)

These services receive only the URL being analyzed. No client IDs, cookies, tokens, or any identifiable data are ever transmitted to external services.

PhishingGuard does not use cookies, trackers, analytics tools, or advertising technologies.

5. Local Processing

Some processing occurs locally within the browser, including:

  • Client ID management
  • Basic UI functions
  • Domain extraction and parsing
  • Rendering warnings and user interface behavior

No sensitive data is processed locally beyond what is necessary to perform these functions.

6. Policy Updates

Any changes to this Privacy Policy will be reflected in the “Last Updated” date above.
